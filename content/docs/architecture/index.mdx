This section explains the technical boundaries that matter when you are changing or debugging BlockMind. Read it as a map: start with the platform and runtime pages to orient yourself, then move into the critical flows for chat, provisioning, credits, durable streams, and authentication.

The important correction in this rewrite is that the most failure-prone pages now match production reality. The relay boundary is no longer described as an oRPC flow, the credit system is no longer described as model-tier billing, and provisioning now follows the real Inngest step order.

## Recommended reading order

1. [Platform](/docs/architecture/platform) for the request surface and endpoint boundaries.
2. [Elite runtime](/docs/architecture/elite-runtime) for where agent work actually runs.
3. [Chat flow](/docs/architecture/chat-flow) for the relay, ingest, and browser event split.
4. [Provisioning](/docs/architecture/provisioning) for workspace lifecycle creation.
5. [Credits](/docs/architecture/credits) and [Durable streams](/docs/architecture/durable-streams) for the two shared cross-cutting systems.
6. [Auth](/docs/architecture/auth) for the human and machine auth surfaces.
