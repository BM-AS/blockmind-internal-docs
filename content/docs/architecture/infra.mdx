## Agent Infrastructure

The infrastructure layer manages the full lifecycle of Elite agent machines — from provisioning to template updates to health monitoring.

<D2Diagram name="AgentInfraContainers" title="Agent Infrastructure — Container View" />

### Containers

| Container | Technology | Purpose |
|-----------|-----------|---------|
| **Provisioner** | Inngest Workflow | Multi-step durable function: DB record → pool claim → key gen → file seeding → activation |
| **Template Sync** | TypeScript | Distributes skill updates + config patches to all active machines |
| **Health Monitor** | TypeScript | Machine status tracking, gateway readiness checks, lifecycle transitions |
| **Runtime Revisions** | TypeScript | Version tracking for OpenClaw image + config schema |
| **Machine Pool** | TypeScript | Pre-warmed Fly Machines for fast provisioning (~200ms claim vs ~30s create) |

### Workspace Status Lifecycle

```
provisioning → hatching → active → suspended → degraded → failed
                                    ↑                      ↓
                                    └──────────────────────┘
                                         (recovery)
```

| Status | Meaning |
|--------|---------|
| `provisioning` | Inngest workflow in progress |
| `hatching` | Machine running, identity sync in progress |
| `active` | Ready for chat |
| `suspended` | Auto-suspended (idle). Resumes in ~200ms. |
| `degraded` | Machine up but gateway unhealthy |
| `failed` | Provisioning or runtime failure |

### Machine Pool

The pool maintains pre-warmed Fly Machines with the base OpenClaw image + volume already attached. When a user provisions:

1. **Fast path (~200ms):** Claim from pool, personalize config, set secrets → active
2. **Slow path (~30s):** Create fresh Fly app + volume + machine from scratch

Pool replenishment runs on a schedule to maintain target count.

### Template Sync

Managed template sync ensures all active machines run the latest skills and configuration:

- **Golden Manifest** — Canonical workspace file set (skills, config, templates) versioned as `MANAGED_TEMPLATE_VERSION`
- **Version Tracking** — Each workspace stores its last-synced template version in config
- **File Writing** — Updates delivered via Fly Machine exec API (`writeWorkspaceFiles`)
- **Throttled** — Won't re-sync within 5 minutes of last sync

### Runtime Revisions

Tracks which version of the OpenClaw image and config schema each workspace runs:

- New revisions are created when the platform ships agent updates
- Pending revisions are reconciled when a machine is started for chat
- Ensures smooth rolling updates without disrupting active sessions
