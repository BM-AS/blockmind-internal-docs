## Credit System

BlockMind uses a credit-based billing model. Each chat run costs credits based on the model tier used.

<D2Diagram name="CreditFlow" title="Credit Flow — Classification → Deduction" />

### Model Tier Pricing

| Tier | Models | Cost per Run |
|------|--------|-------------|
| **Flash** | Gemini Flash, Flash Lite | 0.25 credits |
| **Sonnet** | Claude Sonnet | 1 credit |
| **Opus** | Claude Opus | 3 credits |

### Flow

1. **User sends message** → `chat.send()` via oRPC
2. **Classifier** determines model tier from the agent's configured model
3. **Balance check** — reads `credit_balances` for the user's org
4. **Insufficient?** → Reject with upgrade prompt
5. **Sufficient?** → Proceed to agent
6. **Post-completion** → Deduct credits from balance

### Monthly Renewal

An Inngest cron (`grant-monthly-credits`) tops up credit balances monthly based on subscription tier.

### Key Design Decisions

- **Pre-flight check** — Credits are checked before the run starts, not after. Prevents negative balances.
- **Per-org balances** — Credits belong to organizations, not individual users. Shared pool.
- **Deduction after completion** — Actual cost is deducted post-run, not reserved upfront (avoids phantom deductions on failed runs).
