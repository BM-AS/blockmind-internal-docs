The Elite runtime is where agent work actually happens. The platform starts the work, but the runtime owns execution, upstream gateway interaction, progress emission over the ingest callback stream, and the terminal callbacks that return run state to the platform.

If the platform is the control plane, the runtime is the execution plane. This page stays at that boundary and leaves pricing, provisioning, and auth policy to the dedicated pages that own them.

## Responsibilities

- Execute the run after the platform accepts user work.
- Emit structured callbacks to the platform through the ingest path.
- Launch with separate browser-events and artifact callback URLs in the relay payload.
- Return ingest events that the platform transforms into durable stream events for the product experience.

## Runtime-to-platform contract

The relay launches with:

- `ingestPath: '/api/elite/ingest'`
- a separate `browserEventsUrl`
- a separate `artifactCreateUrl` (defaulting to `/api/agent/artifacts`)

The platform also exposes `/api/elite/artifacts/...` as the authenticated artifact access route for product reads. That route is distinct from the relay's artifact upload callback target.

That split exists in the launch contract today. The ingest path is active, while the browser-events and artifact callback plumbing is configured separately but is not currently emitted by the relay implementation.

## Lifecycle in practice

1. The platform accepts a chat request on `/api/elite/chat`.
2. The runtime starts executing the run.
3. The relay emits ingest events on `/api/elite/ingest`; browser-events and artifact callback URLs are configured separately but are not currently used by the relay implementation.
4. The platform transforms ingest callbacks into durable stream events for the product experience.

## Related operational state

The runtime only becomes useful after the provisioning workflow moves a workspace into an operational state (`active` or `degraded`). Later operational transitions such as `suspended` and `failed` affect whether the platform can hand new work to the runtime cleanly.

- Read [Workspace states](/docs/operations/workspace-states) for the operational lifecycle.
